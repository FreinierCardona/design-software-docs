# Producto

> Estado: 🟢 Completo | Última actualización: 2026-06-29
> Autor: Equipo de Producto | Equipo: Product Management

## Propósito

Esta sección centraliza la documentación relacionada con la visión funcional del Sistema de Gestión de Horarios SENA desde la perspectiva del producto.

Su objetivo es definir qué problema resuelve el sistema, cuál es su propuesta de valor, cómo evolucionará en el tiempo y cuáles son las funcionalidades priorizadas que guiarán el desarrollo del proyecto.

La documentación aquí contenida sirve como referencia para Product Owners, Arquitectos, Analistas, Scrum Masters y equipos de desarrollo, asegurando que todos trabajen sobre una misma estrategia de producto.

## Contenido

La documentación del producto está organizada en tres componentes principales:

- **Visión del producto:** define el propósito del sistema, objetivos estratégicos, alcance funcional y propuesta de valor para los diferentes actores involucrados.
- **Roadmap:** presenta la planificación de alto nivel de la evolución del producto mediante fases e hitos de implementación.
- **Product Backlog:** consolida las iniciativas, épicas e historias de usuario priorizadas que representan el trabajo funcional del producto.

## Relación con el proyecto

La información de esta carpeta mantiene alineación directa con:

- Arquitectura de Microservicios.
- Clean Architecture by Module.
- Domain Driven Design (DDD).
- Bounded Contexts definidos para el Sistema de Gestión de Horarios SENA.
- Objetivos funcionales establecidos para instructores, aprendices, coordinadores académicos y administradores.
