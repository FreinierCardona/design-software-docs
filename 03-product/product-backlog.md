# Product Backlog

> Estado: 🟢 Completo | Última actualización: 2026-06-29
> Autor: Equipo de Producto | Equipo: Product Management

## Propósito

Este documento describe la estrategia de gestión del Product Backlog del Sistema de Gestión de Horarios SENA.

El detalle de las épicas, historias de usuario, tareas, prioridades y estados de desarrollo no se mantiene en este repositorio, sino en la herramienta de gestión ágil adoptada por el equipo.

## Herramienta de gestión

El Product Backlog oficial se administra mediante **Trello**, permitiendo el seguimiento continuo del trabajo, la priorización de funcionalidades y el avance de cada Sprint.

## Flujo de trabajo

Las tarjetas del backlog siguen el siguiente flujo de estados:

| Estado | Descripción |
|---------|-------------|
| **Backlog** | Funcionalidades identificadas pendientes de priorización o planificación. |
| **To Do** | Historias de usuario seleccionadas para ser desarrolladas próximamente. |
| **In Progress** | Trabajo que actualmente está siendo desarrollado por el equipo. |
| **Done** | Funcionalidades completamente implementadas, validadas y finalizadas. |

## Organización del backlog

Las funcionalidades se organizan mediante una jerarquía compuesta por:

- Épicas.
- Historias de Usuario (HU).
- Tareas técnicas.
- Bugs o incidencias.
- Mejoras evolutivas.

## Priorización

La priorización del Product Backlog se realiza considerando:

- Valor para el negocio.
- Dependencias funcionales y técnicas.
- Riesgos de implementación.
- Complejidad estimada.
- Objetivos definidos para cada Sprint.

## Relación con la documentación

La documentación contenida en este repositorio describe la arquitectura, el dominio, el diseño y las decisiones técnicas del proyecto.

El Product Backlog representa la planificación dinámica del desarrollo y constituye la fuente oficial para el seguimiento del trabajo del equipo.
