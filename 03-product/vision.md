# Propósito

El Sistema de Gestión de Horarios SENA es una plataforma orientada a optimizar la planificación, asignación y seguimiento de los horarios académicos dentro de los centros de formación del SENA.

El producto busca centralizar la gestión académica y operativa mediante una solución moderna, escalable y modular que permita automatizar procesos, reducir conflictos de programación y facilitar la toma de decisiones basada en información confiable.

---

# Problema

La gestión manual o parcialmente digitalizada de los horarios presenta diferentes dificultades:

- Conflictos en la asignación de instructores.
- Doble reserva de ambientes de formación.
- Procesos administrativos repetitivos.
- Baja trazabilidad de los cambios realizados.
- Escasa visibilidad del estado de la programación académica.
- Dificultad para mantener información consistente entre las diferentes áreas involucradas.

Estas situaciones generan retrasos operativos, reprocesos y afectan la calidad de la planeación académica.

---

# Visión del producto

Desarrollar una plataforma institucional que permita administrar de manera integral la programación académica del SENA mediante procesos automatizados, garantizando consistencia, disponibilidad de recursos, trazabilidad de las operaciones y escalabilidad para futuras necesidades institucionales.

---

# Objetivos

## Objetivo general

Construir un sistema que permita gestionar de forma eficiente la programación de horarios académicos, recursos físicos y actores involucrados en los procesos de formación.

## Objetivos específicos

- Centralizar la información académica y administrativa.
- Automatizar la asignación de horarios.
- Detectar conflictos antes de publicar la programación.
- Gestionar ambientes, instructores y fichas de formación.
- Mantener trazabilidad completa sobre las modificaciones realizadas.
- Facilitar el monitoreo mediante indicadores y alertas.
- Permitir la evolución continua del sistema mediante una arquitectura desacoplada.

---

# Usuarios del producto

El sistema está diseñado para atender las necesidades de diferentes actores institucionales.

| Usuario | Necesidad principal |
|----------|---------------------|
| Administrador | Configurar y administrar el sistema. |
| Coordinador académico | Planificar y controlar la programación académica. |
| Instructor | Consultar horarios y asignaciones. |
| Aprendiz | Consultar su horario de formación. |
| Personal administrativo | Gestionar ambientes, documentos y recursos institucionales. |

---

# Propuesta de valor

El Sistema de Gestión de Horarios SENA ofrece una solución que:

- Centraliza toda la gestión académica.
- Reduce errores de programación.
- Optimiza el uso de ambientes e instructores.
- Disminuye tiempos administrativos.
- Facilita la trazabilidad y auditoría de cambios.
- Permite escalar funcionalmente mediante arquitectura de microservicios.
- Proporciona información confiable para la toma de decisiones.

---

# Capacidades principales

El producto integra funcionalidades distribuidas en diferentes dominios del negocio:

- Gestión de usuarios y seguridad.
- Gestión académica.
- Gestión de ambientes.
- Programación de horarios.
- Gestión documental.
- Seguimiento académico.
- Auditoría.
- Notificaciones y alertas.
- Administración de parámetros institucionales.

---

# Principios del producto

El desarrollo del sistema se fundamenta en los siguientes principios:

- Arquitectura orientada a microservicios.
- Clean Architecture por módulos.
- Domain-Driven Design (DDD).
- Separación de responsabilidades.
- Escalabilidad.
- Seguridad desde el diseño.
- Trazabilidad de las operaciones.
- Mantenibilidad y evolución continua.

---

# Estado actual

Actualmente el producto se encuentra en fase de definición arquitectónica y planificación funcional, estableciendo la base documental, el modelo de dominio, la arquitectura de software y la organización del desarrollo que soportarán la construcción progresiva del sistema.
