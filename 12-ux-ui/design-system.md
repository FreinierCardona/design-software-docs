# Design System

> Estado: 🟢 Completo | Última actualización: 2026-06-30
> Autor: Equipo UX/UI
> Equipo: Diseño de Experiencia e Interfaces

## Propósito

Este documento establece los lineamientos del Sistema de Diseño (Design System) del Sistema de Gestión de Horarios SENA.

Su objetivo es definir los principios visuales, componentes reutilizables y reglas de interacción que garanticen una experiencia consistente, accesible y uniforme en todas las interfaces del proyecto, independientemente de la plataforma o del equipo responsable de su desarrollo.

El Sistema de Diseño constituye la referencia oficial para el diseño y construcción de interfaces de usuario del sistema.

---

# Objetivos

El Sistema de Diseño busca:

- Garantizar consistencia visual en todas las aplicaciones.
- Facilitar la reutilización de componentes.
- Reducir inconsistencias entre equipos.
- Mejorar la experiencia del usuario.
- Simplificar el mantenimiento de las interfaces.
- Favorecer la escalabilidad del producto.
- Promover accesibilidad y usabilidad.

---

# Alcance

El Sistema de Diseño documenta los lineamientos relacionados con:

- Identidad visual.
- Componentes reutilizables.
- Patrones de interacción.
- Jerarquía visual.
- Iconografía.
- Tipografía.
- Colores institucionales.
- Espaciados.
- Accesibilidad.
- Comportamiento general de la interfaz.

No incluye implementaciones en código, bibliotecas de componentes ni archivos de diseño editables.

---

# Principios de diseño

Todas las interfaces deberán construirse siguiendo los siguientes principios:

- Consistencia.
- Simplicidad.
- Claridad.
- Accesibilidad.
- Reutilización.
- Escalabilidad.
- Diseño centrado en el usuario.
- Coherencia entre plataformas.

---

# Identidad visual

La documentación deberá definir los lineamientos generales relacionados con:

- Paleta de colores institucional.
- Tipografía oficial.
- Iconografía.
- Espaciado.
- Jerarquía visual.
- Uso de logotipos.
- Elementos gráficos comunes.

Toda modificación deberá mantenerse alineada con la identidad institucional del proyecto.

---

# Componentes

El Sistema de Diseño deberá documentar los componentes reutilizables utilizados por las interfaces.

Para cada componente se recomienda registrar:

- Nombre.
- Propósito.
- Descripción funcional.
- Variantes disponibles.
- Estados.
- Restricciones de uso.
- Casos de aplicación.

La implementación técnica de los componentes deberá mantenerse en los proyectos correspondientes.

---

# Patrones de interacción

La documentación deberá establecer criterios comunes para las interacciones del usuario, incluyendo cuando aplique:

- Navegación.
- Formularios.
- Confirmaciones.
- Validaciones.
- Mensajes del sistema.
- Retroalimentación visual.
- Manejo de errores.
- Estados de carga.

---

# Accesibilidad

Todas las interfaces deberán considerar buenas prácticas de accesibilidad, promoviendo:

- Contraste adecuado.
- Navegación mediante teclado cuando aplique.
- Etiquetas descriptivas.
- Jerarquía correcta de contenidos.
- Componentes comprensibles.
- Experiencia inclusiva para diferentes perfiles de usuario.

---

# Adaptabilidad

El Sistema de Diseño deberá contemplar lineamientos para mantener consistencia entre:

- Aplicación Web.
- Aplicación Móvil Android.

Cada plataforma podrá adaptar sus componentes respetando las convenciones propias del entorno, sin perder la identidad visual ni la coherencia funcional del sistema.

---

# Gobernanza

Toda incorporación o modificación de componentes deberá:

- responder a una necesidad funcional;
- evitar la duplicación de componentes existentes;
- mantenerse documentada;
- ser validada por el equipo UX/UI;
- conservar compatibilidad con el resto del Sistema de Diseño.

# Mantenimiento

El Sistema de Diseño deberá actualizarse cuando exista alguno de los siguientes cambios:

- incorporación de nuevos componentes;
- modificación de patrones visuales;
- actualización de la identidad gráfica;
- evolución de la experiencia de usuario;
- mejoras de accesibilidad;
- incorporación de nuevas plataformas.

Toda actualización deberá mantenerse alineada con la visión del producto, la arquitectura y los objetivos funcionales del Sistema de Gestión de Horarios SENA.
