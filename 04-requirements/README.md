# Requisitos

> Estado: 🟢 Completo | Última actualización: 2026-06-29
> Autor: Equipo de Arquitectura | Equipo: Arquitectura de Software

## Propósito

Este módulo documenta todos los requisitos del Sistema de Gestión de Horarios SENA (SGH SENA), estableciendo las necesidades funcionales y no funcionales que deberá satisfacer la solución.

La documentación aquí contenida constituye la base para el análisis, diseño, desarrollo, pruebas y validación del sistema, garantizando que todas las decisiones técnicas se encuentren alineadas con los objetivos del negocio y las necesidades de los diferentes actores del SENA.

Los requisitos definidos en esta sección mantienen trazabilidad con las historias de usuario, el backlog del producto, la arquitectura del sistema, los casos de uso, los módulos funcionales y los escenarios de prueba.

---

# Objetivos

Los documentos de este módulo permiten:

- Definir el comportamiento esperado del sistema.
- Especificar los atributos de calidad del software.
- Documentar las necesidades de cada actor del sistema.
- Mantener trazabilidad entre requisitos, historias de usuario y pruebas.
- Facilitar el mantenimiento y evolución del proyecto.
- Servir como referencia durante todo el ciclo de vida del desarrollo.

---

# Alcance

La documentación de requisitos comprende:

- Requisitos funcionales.
- Requisitos no funcionales.
- Historias de usuario.
- Criterios de aceptación.
- Matriz de trazabilidad.
- Relación con backlog y planificación del proyecto.

No incluye documentación de arquitectura, diseño técnico, modelos de datos o procedimientos de despliegue, los cuales se encuentran documentados en sus respectivos módulos del repositorio.
