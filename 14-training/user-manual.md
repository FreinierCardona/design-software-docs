# User Manual

> Estado: 🟢 Completo | Última actualización: 2026-06-30
> Autor: Equipo de Producto
> Equipo: Producto y Soporte

## Propósito

Este documento proporciona la guía de referencia para los usuarios finales del Sistema de Gestión de Horarios SENA.

Su objetivo es facilitar el uso adecuado de la plataforma, describiendo los conceptos generales, las funcionalidades disponibles y las buenas prácticas que deben seguir los diferentes perfiles de usuario durante su interacción con el sistema.

El manual constituye la referencia principal para la capacitación y consulta de aprendices, instructores, coordinadores académicos y demás actores autorizados.

---

# Objetivos

El manual busca:

- Facilitar el aprendizaje del sistema.
- Estandarizar el uso de las funcionalidades.
- Reducir errores durante la operación.
- Servir como material de consulta permanente.
- Mejorar la experiencia del usuario.
- Apoyar los procesos de capacitación y soporte.

---

# Alcance

Este manual documenta los aspectos funcionales relacionados con el uso del sistema, incluyendo:

- Acceso a la plataforma.
- Navegación general.
- Gestión del perfil de usuario.
- Consulta de información.
- Uso de las funcionalidades autorizadas.
- Buenas prácticas de utilización.

No incluye procedimientos administrativos, configuración técnica, despliegue, desarrollo de software ni mantenimiento de infraestructura.

---

# Usuarios del sistema

El Sistema de Gestión de Horarios SENA está diseñado para atender diferentes perfiles de usuario, entre ellos:

- Aprendices.
- Instructores.
- Coordinadores académicos.
- Personal administrativo.
- Administradores funcionales.

Las funcionalidades disponibles dependerán del rol y los permisos asignados a cada usuario.

---

# Acceso a la plataforma

Para utilizar el sistema, cada usuario deberá:

- disponer de una cuenta autorizada;
- autenticarse mediante los mecanismos definidos por la organización;
- utilizar únicamente las funcionalidades permitidas para su perfil.

El acceso a la información estará controlado por los permisos asignados a cada rol.

---

# Navegación general

La plataforma organiza sus funcionalidades por módulos funcionales, permitiendo al usuario acceder únicamente a las opciones correspondientes a sus responsabilidades.

La estructura de navegación busca ofrecer una experiencia:

- clara;
- consistente;
- intuitiva;
- orientada a las tareas del usuario.

---

# Funcionalidades

Dependiendo del perfil asignado, el usuario podrá acceder a funcionalidades relacionadas con:

- Consulta de horarios.
- Gestión de asignaciones.
- Consulta de ambientes.
- Información académica.
- Notificaciones.
- Gestión de documentos.
- Seguimiento de actividades.
- Actualización de información personal.

Las funcionalidades disponibles podrán evolucionar conforme avance el desarrollo del sistema.

---

# Buenas prácticas

Se recomienda a todos los usuarios:

- utilizar únicamente sus credenciales personales;
- mantener actualizada su información cuando corresponda;
- verificar la información antes de registrar cambios;
- cerrar la sesión al finalizar el uso del sistema;
- reportar comportamientos inesperados al equipo de soporte;
- respetar las políticas institucionales de uso de la plataforma.

---

# Soporte al usuario

Cuando un usuario presente dificultades durante la utilización del sistema, deberá:

1. Verificar la información ingresada.
2. Confirmar que dispone de los permisos necesarios.
3. Revisar la documentación disponible.
4. Contactar al equipo de soporte cuando el problema persista.

Las incidencias técnicas serán atendidas conforme a los procedimientos definidos por la organización.
